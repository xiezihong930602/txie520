# TEMU全托管自动上架RPA系统 V2

## 一、项目概述

基于妙手ERP的TEMU全托管店铺自动上架RPA系统，采用「飞书多维表格（数据中台）+ Python RPA（执行引擎）」架构。

- **店铺**：Noble Boys（店铺ID 14255939，平台pddkj）
- **核心能力**：自动读取款式库数据 → RPA模拟人工填写妙手后台 → 批量创建商品
- **核价系数**：默认2倍（供货价 = 成本价 × 2）


## 六、核心计算公式

### 6.1 价格
- **供货价** = 成本价 × 核价系数（默认2）
- **建议售价** = 供货价 × 2.5

### 6.2 尺寸（包装）
- 基础值（1件）：长25cm × 宽20cm × 高3cm
- 每增加1件：长+3cm、宽+3cm、高+1cm
- 例：4件装 → 34×29×6 cm

### 6.3 重量
- **公式**：最终重量 = 单码净重 × 件数 + 10g + 10g × 件数
- 例：1件150g → 150×1 + 10 + 10×1 = 170g
- 例：3件120g → 120×3 + 10 + 10×3 = 400g

### 6.4 SKU分类三种模式
| 模式 | 分类 | 数量 | 单位 | 共计内含 | 是否独立包装 |
|------|------|------|------|----------|--------------|
| 单品 | 单品 | 1 | 件 | 1 | - |
| 同款多件装 | 同款多件装 | 实际件数 | 件 | 实际件数 | 不是独立包装 |
| 混合套装 | 混合套装 | 总件数 | 件 | - | 不是独立包装 |

## 八、技术说明

### 8.1 登录方案
- Playwright `storage_state` 机制，保存Cookie和LocalStorage
- 登录态文件：`storage_state.json`
- 过期后重新运行 `python login_browser.py`

### 8.2 图片上传方案
- 使用网络图片URL方式上传，绕过本地文件选择器的超时问题
- 必须取消勾选「同时保存到妙手图片空间」
- 多图URL用换行分隔，顺序即展示顺序

### 8.3 运行环境
- Python 3.12
- Playwright 1.61.0
- 浏览器驱动：项目内 `browser_drivers/` 目录
- **必须本地cmd运行**，沙箱环境无法启动Playwright

### 8.4 测试模式参数
- `auto_close=False`：跑完不自动关闭浏览器，方便手动检查
- `auto_submit=False`：不自动点「确定创建」，人工确认后再提交

## 十、注意事项

1. **模板名称必须准确**：不同模板对应不同的尺码表列表，模板选错了找不到对应尺码表
2. **款式名称必须与款式库一致**：作为关联Key，错了读不到尺码和重量数据
3. **产地不要改**：属于模板固定属性，RPA不修改
4. **每个SKC至少5张图**：TEMU平台规则
5. **测试模式不自动提交**：确认填写无误后手动点「确定创建」
6. **登录态定期更新**：storage_state约1-2周过期，需重新登录
7. **CSV文件编码**：`测试数据.csv` 是GBK编码，Excel直接保存即可
8. **重量数据来源**：从款式库Excel的「尺码明细」Sheet读取每个尺码的净重
