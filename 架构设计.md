# TEMU自动上架系统 V2 - 架构设计

## 整体架构
配置驱动 + 模块化执行器 + 款式库复用

## 目录结构
```
temu_auto_publish_v2/
├── config/          # 配置文件
│   ├── settings.py  # 全局配置
│   └── templates.py # 品类模板映射
├── data_source/     # 数据源模块
│   └── feishu.py    # 飞书多维表格读写
├── models/          # 数据模型
│   ├── product.py   # 商品数据结构
│   └── style.py     # 款式数据结构
├── executors/       # 执行器模块（可插拔）
│   ├── base.py      # 执行器基类
│   ├── rpa_publisher.py  # RPA上架执行器
│   └── image_generator.py # 生图执行器（预留）
├── workflow/        # 工作流引擎
│   └── engine.py    # 工作流调度
├── utils/           # 工具函数
└── main.py          # 入口程序
```

## 飞书多维表格设计（3张表）

### 表1：款式库表（基础数据，维护一次）
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 款式编号 | 单行文本 | 是 | 唯一标识，如 POLO-001 |
| 款式名称 | 单行文本 | 是 | 华夫格短袖Polo衫 |
| 品类模板 | 单行文本 | 是 | 对应妙手后台的模板名称 |
| 面料 | 单选 | 是 | 下拉选项：聚酯纤维/棉/混纺等 |
| 季节 | 单选 | 是 | 春夏/秋冬/四季 |
| 版型 | 单选 | 是 | 标准/修身/宽松 |
| 织造方式 | 单选 | 是 | 针织/梭织/华夫格等 |
| 场景 | 单选 | 是 | 运动/休闲/居家 |
| 场合 | 单选 | 是 | 日常/运动/出游 |
| 单件净重(g) | 数字 | 是 | 单件重量，用于计算 |
| 单件毛重(g) | 数字 | 是 | 含包装重量 |
| 默认SKU分类 | 单选 | 是 | 单品/2件装/套装 |
| 成本价(元) | 数字 | 是 | 用于核价：售价=成本×2 |
| 尺码列表 | 多行文本 | 是 | 逗号分隔，如 130,140,150 |
| 备注 | 多行文本 | 否 | |

### 表2：商品上架表（业务主表，每行一个商品）
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 商品编号 | 单行文本 | 是 | 唯一ID |
| 主款式编号 | 关联款式库 | 是 | 自动带出所有属性 |
| 组合类型 | 单选 | 是 | 单件/2件同款/上下装套装/3件套 |
| 副款式编号 | 关联款式库 | 否 | 套装的下装款式 |
| 商品标题(英文) | 单行文本 | 是 | TEMU商品标题 |
| 主图URL列表 | 多行文本 | 是 | 每行一个URL，至少5张 |
| 颜色列表 | 多行文本 | 是 | 每行一个颜色名称 |
| 尺码列表 | 多行文本 | 否 | 不填则用款式库默认 |
| 供货价(元) | 数字 | 否 | 不填则自动计算：成本×2 |
| 净重覆盖(g) | 数字 | 否 | 不填则自动计算组合重量 |
| 毛重覆盖(g) | 数字 | 否 | 不填则自动计算 |
| SKU分类覆盖 | 单选 | 否 | 不填则用组合类型对应 |
| 执行状态 | 单选 | 自动 | 待执行/执行中/成功/失败 |
| SKC ID | 单行文本 | 自动 | 上架成功后回填 |
| 错误信息 | 多行文本 | 自动 | 失败原因 |
| 创建时间 | 日期 | 自动 | |

### 表3：模板配置表（系统配置，维护一次）
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 模板名称 | 单行文本 | 是 | 妙手后台的模板名称 |
| 对应类目 | 单行文本 | 是 | 类目路径 |
| 变量字段清单 | 多行文本 | 是 | 需要覆盖的属性字段，逗号分隔 |
| 备注 | 多行文本 | 否 | |

## 核价与重量计算规则
1. 供货价 = 成本价 × 2（可手动覆盖）
2. 单件：净重=单件净重，毛重=单件毛重
3. 2件同款：净重=单件净重×2，毛重=单件毛重×2
4. 上下装套装：净重=上装净重+下装净重，毛重=上装毛重+下装毛重
5. 所有计算值都支持手动覆盖

## 执行器接口规范
所有执行器继承BaseExecutor，实现execute方法：
- 输入：Product对象
- 输出：执行结果（成功/失败 + 数据 + 错误信息）
